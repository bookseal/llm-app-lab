Greenfield (새로 시작)

  • 레포 분석 subagent 탐색기 · 파일 변경 감지 에이전트

워크숍은 이미 배운 조각의 조합이에요. 채팅 UI/메모리(M2), 도구 호출 루프(M3·M5), 그리고 위험 도구 제한 같은 안전(M5·M6)이 하나로 모입니다. 새 개념이 아니라 통합이죠.

7.4 · Final Quiz 슬라이드 4–8

통과 암호로 잠긴 2세트 — 전 모듈을 가로지르는 10문항. 아래는 그 예열용 복습 퀴즈예요.

  • 1 of 2 — Foundations → Tools → Context
  • 2 of 2 — Context → Agents → Production

주제: API statefulness, hallucination, structured outputs, MCP 아키텍처, RAG 검색, 시맨틱 검색, agent loop, subagents, 실패 모드, 테스트 전략.

[복습] LLM API가 "stateless"라는 말의 뜻은?

stateless = 요청 간 기억 없음이에요(M2). 그래서 대화 히스토리를 매 턴 다시 보내거나 (단기 메모리), 사실을 벡터 스토어에 저장해 검색(M4·M5의 long-term memory)합니다.

[복습] RAG에서 인용 [n]을 강제하는 핵심 이유는?

인용은 귀속 강제 = hallucination 방어예요(M4). 받은 답의 번호가 실제 청크 범위 안인지 코드로 한 번 더 검증해 환각을 거릅니다.

[복습] 에이전트의 폭주(무한 루프·비용 폭탄)를 막는 1차 방어선은?

자율성과 안전은 트레이드오프(M5)예요. 모델을 믿기보다 하니스가 강제하는 한도가 폭주를 막고, 비가역 행동엔 사람 확인을 끼웁니다.
