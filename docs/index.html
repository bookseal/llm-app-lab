Build with LLMs

Seven hands-on modules. You don't just read about LLM apps — you build four small ones, one idea at a time, then assemble your own.

KSEPT Summer Program · Building with LLMs. Each module pairs a short concept note with a starter you run on your own machine. By the end you can take an LLM from "a chat box" to "an app that searches your documents and acts on a database."

🌏 Note on language: the concept notes are written in Korean so the ideas land clearly, but every diagram, code sample, and project is in English. New to LLMs? Start at Module 1 and go in order — each page builds on the one before.
🛠️ Already built a web app with a frontend, a backend, and a login screen? Then you've done the hard parts. This is the LLM version of what you already know — same two-server shape, plus one new skill per module.

💬 Module 2 · chat-app

Talk to Claude — a real chat app

A streaming chat with conversation memory, loading states, and honest error handling. You'll learn the one fact that trips up every beginner: the API is stateless — it forgets you between requests, so you carry the memory.

🧩 Module 3 · job-post extractor

Turn messy text into clean data

Point Claude at a free-text job posting and get back validated JSON — title, pay, location, requirements. The trick is tool use: you force the model to fill a schema instead of writing prose, then validate before you trust it.

📚 Module 4 · RAG app

Answer questions over your own documents

Index a small corpus, retrieve only the relevant chunks, and answer with citations you can check. This is how you give an LLM knowledge it was never trained on — without stuffing a whole library into the prompt.

🤖 Module 5 · data-analyst agent

Ask in English, get answers from a database

A ~20-line agent loop calls read-only SQL tools, reads the rows, and explains the result in plain language. This same little loop is the engine inside Claude Code, Cursor, and every "agent" you've heard about.

Module by module — learn & build

Module 1 · Setup

Get your toolchain ready

Learn: VSCode · Git · Node.js · Python 3.11 · Claude Code CLI · an API key.

Build: a working dev environment with one shared .env every example reuses.

Module 2 · Foundations

How an LLM app is actually wired

Learn: two servers + a proxy, JSX, mount, and why the API is stateless.

Build: in two parts —

  • 2-1 · Foundations (hello-world) — Flask + React, the plumbing.
  • 2-2 · chat-app (project) — memory, loading, network & API errors, "find the 5 bugs".
Module 3 · Tools & Structure

Make the model's output machine-usable

Learn: 4 levels — parseable text → JSON schema → tool use → MCP.

Build: a job-post extractor that returns validated, structured fields.

Module 4 · Context / RAG

Give the model knowledge it wasn't trained on

Learn: embeddings, chunking, vector search (ANN/HNSW), citations, memory.

Build: a RAG app that answers over a corpus and cites its sources.

Module 5 · Agents

The engine behind every agent

Learn: the ~20-line agent loop, subagents, memory scopes, computer use, safety.

Build: a data-analyst agent over a read-only SQL database.

Module 6 · Production

Make it trustworthy and operable

Learn: the eval ladder, prompt-injection defense, observability & latency.

Build: guardrails, evals, and telemetry around the app you already made.

Module 7 · Workshop

Build your own — the build day

Learn: nothing new — it's all assembly now.

Build: extend one of your projects, or start something fresh. Small but shippable.
