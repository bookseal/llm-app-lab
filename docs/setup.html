Module 1 · Setup

수업 전 개발 환경을 갖춥니다. 6가지 도구를 설치하고, 마지막에 API 키 하나를 공유 .env에 저장하면 코스의 모든 예제가 그걸 자동으로 찾아 씁니다.

📊 원본 슬라이드: ksetp.netlify.app/setup — 각 섹션의 칩(슬라이드 N)이 덱의 슬라이드 번호와 같습니다.
🧰 비유: 요리를 시작하기 전 주방에 칼·도마·불을 갖추는 단계예요. 여기선 안 쓰지만, 이게 없으면 2편부터 아무것도 못 합니다.

Install VSCode 슬라이드 1

코드를 들여다보고 편집하는 에디터(IDE). 필수는 아니지만 곧 필요해집니다.

# macOS
brew install --cask visual-studio-code
# Windows
winget install Microsoft.VisualStudioCode
# Verify
code --version

code 명령이 없으면 — macOS: VSCode → ⌘⇧P → "Shell Command: Install 'code' command in PATH". Windows: 설치 재실행 후 "Add to PATH" 체크.

Install Git 슬라이드 2

변경을 한 커밋씩 추적 → 발표 때 "diff를 따라 걷기". 42에서 매일 쓰던 그 git이에요.

# macOS (보통 Xcode CLT로 이미 있음)
brew install git
# Windows
winget install Git.Git
# Verify
git --version

GitHub 계정/원격은 불필요 — git은 완전히 로컬로 동작합니다. repo는 프로젝트 안의 숨김 .git/ 폴더일 뿐이에요.

Install Node.js (LTS) 슬라이드 3

React 프론트엔드를 빌드·실행하는 도구(npm 포함).

# macOS
brew install node
# Windows
winget install OpenJS.NodeJS.LTS
# Verify
node --version    # v20.x.x or v22.x.x
npm --version

또는 nodejs.org → LTS(v20/v22).

Install Python 3.11 슬라이드 4

Flask 백엔드를 돌리는 언어. 버전을 3.11로 맞춥니다.

# macOS
brew install python@3.11
# Windows
winget install Python.Python.3.11
# Verify
python3.11 --version    # Python 3.11.x
py -3.11 --version      # Windows: py 런처로 특정 버전 선택

Create your project folder and venv 슬라이드 5

프로젝트 폴더를 만들고, 그 안에 격리된 파이썬 환경(venv)을 켭니다.

# macOS
mkdir ~/ksept-lab && cd ~/ksept-lab
python3.11 -m venv .venv
source .venv/bin/activate

# Windows (PowerShell)
mkdir ~/ksept-lab; cd ~/ksept-lab
py -3.11 -m venv .venv
.venv\Scripts\Activate.ps1
📦 비유: venv는 "이 프로젝트 전용 서랍"이에요. 활성화 중 pip install한 건 시스템 전체가 아니라 이 서랍에만 들어갑니다 — 프로젝트끼리 라이브러리가 안 엉켜요.

Windows에서 스크립트가 막히면 한 번만: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned. 확인: which python이 .../ksept-lab/.venv/bin/python으로 끝나야 합니다.

venv를 활성화한 채 pip install flask를 하면?

venv 활성화 = "이 서랍을 기본으로 써"라는 뜻. 그래서 설치물이 격리됩니다. .venv는 git에서 제외돼요(용량 큼, 재생성 가능).

Install the Claude Code CLI 슬라이드 6

앱을 대신 스캐폴딩해 주는 AI 코딩 어시스턴트(지금 당신이 쓰는 그것).

# macOS
npm install -g @anthropic-ai/claude-code
# Windows (자체 바이너리, Node 불필요)
irm https://claude.ai/install.ps1 | iex
# Verify
claude --version
cd ~/ksept-lab
code .     # VSCode로 폴더 열기
claude     # 통합 터미널에서 실행 — 첫 실행 시 로그인 안내

Get an Anthropic API key 슬라이드 7

당신의 앱이 Claude를 호출할 때 쓰는 열쇠. Claude Code 자체는 자기 로그인이 있어 필요 없고, 이 키는 예제 앱용이에요.

  1. console.anthropic.com/settings/keys → Create Key → 지금 복사(다시 안 보임)
  2. 루트의 공유 .env 한 곳에 저장:
# macOS
cd ~/ksept-lab
echo 'ANTHROPIC_API_KEY=sk-ant-...your-key...' > .env

# Windows
'ANTHROPIC_API_KEY=sk-ant-...your-key...' | Out-File -Encoding ascii .env
🔒 핵심: .env는 git에서 제외됩니다. 각 앱보다 한 단계 위 (~/ksept-lab/)에 두면, 모든 예제 앱이 폴더 트리를 거슬러 올라가 이 키 하나를 자동으로 찾아 씁니다 — 키를 여러 번 적을 필요가 없어요.
