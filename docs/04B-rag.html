4B.1 · What it builds

20개 문서(Apollo 코퍼스)에 대해 질문하면 출처 번호가 달린 답을 주는 챗.

하는 일
  • 문서를 청크로 쪼개 임베딩 → 벡터 스토어에 저장(indexing)
  • 질문을 임베딩해 가장 가까운 top-K 청크 검색(retrieve)
  • [n] 번호 블록으로 붙여(augment) Claude 한 번 호출(generate)
  • 코퍼스에 없는 질문(예: Artemis)은 모른다고 거절

4B.3 · Run it live

앱을 빌드해 두면, 버튼이 서버에서 잠깐 띄워 URL을 돌려줍니다.

⚙️ 런처가 rag 이미지를 띄울 수 있을 때 활성화됩니다. 그 전엔 버튼이 아래 빌드 & 로컬 실행으로 안내해요.

4B.4 · Build & run locally

스타터를 받아 인덱싱 → RAG 백엔드를 채우고 실행.

# 1) 스타터 받기 (documents/, indexer.py, RAG 스텁 backend 포함)
#    https://ksetp.netlify.app/assets/context/rag-starter.zip
unzip rag-starter.zip && cd rag-starter
python3.11 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2) 코퍼스 인덱싱 (chunk → embed → store)
python indexer.py documents/

# 3) RAG 백엔드 실행 (retrieve · augment · generate 채워 넣기)
python app.py            # → http://localhost:5000

코퍼스·채점 기준은 rag-contest/CONTEST.md(100점 루브릭, 5개 질문, Apollo 20문서)를 보세요.

4B.5 · The key idea — retrieve, then ground

모델에 도서관을 통째로 주지 말고, 관련 몇 페이지만 골라 인용을 강제.

코퍼스에 없는 걸 물으면 좋은 RAG 앱은?

인용 강제 = 근거 없으면 답하지 말 것이에요. 받은 답의 [n]이 실제 청크 범위 안인지 코드로 한 번 더 검증해 환각을 거릅니다(4A.7).
